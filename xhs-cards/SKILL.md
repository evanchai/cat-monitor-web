# SKILL: XHS Vibe Coding 日记卡片生成器
> ningcode design system · 面向非技术受众的 vibe coding 分享

## 一句话定位

把一个 vibe coding 项目，拆成 6 张小红书图文卡，用讲故事的方式呈现，受众是不会写代码的普通人。

## 触发条件

用户说以下任意一种时执行本 skill：
- "帮我做成小红书"
- "生成 xhs 卡片"
- "vibe coding 日记"
- "给这个项目做图文"

## 语气原则（最重要）

不要说：两级检测架构 → 要说：像个永不睡觉的保安
不要说：置信度 0.7 → 要说：摄像头自己判断有没有猫
不要说：Redis TTL → 要说：断电了会自动知道
不要说：边缘计算 → 要说：猫的照片一张都没上传到网上

核心原则：把技术细节翻译成「对我的生活有什么用」。

## 封面标题公式

好标题 = 具体事件（最好有点好笑）+ 意外解决方式 + 疑问句

示例：
- 小猫又尿沙发了，我 vibe 了一个家庭智能摄像头？
- 我用 AI 帮我回复了一个月微信，没人发现？
- 花了 3 小时，我把支出做成了一个会说话的图表

副标题固定模式：不会写代码没关系——我也是边问 AI 边做出来的。

## 各页内容提炼指南

### 01 封面
- 标题：口语化，有具体生活场景，带疑问/悬念
- 副标题：1-2 句，强调不需要会写代码
- 今日主角：项目里的具体对象

### 02 起因
- 一个具体的生活问题，不是"我想学 AI"
- 一句 quote，总结问题本质
- 说清楚为什么不用现成方案

### 03 用了啥
- 把技术栈翻译成 3 样工具
- 每样给一个比喻或功能描述，不写型号参数

### 04 怎么运作
- 找一个比喻句作为标题
- 用 checklist 列出 4 个步骤，每步骤不超过 20 字

### 05 效果
- 一个具体的第一次成功场景
- 3 格数字（时间/费用/频次）
- 一句 quote，说最重要的感受

### 06 收尾
- 鼓励不会写代码没关系
- 列出用到的工具
- 预告下期 + 引导关注 ning.codes

卡片尺寸：390×553px

## 执行步骤

1. 读取本文件 `SKILL.md`
2. 读取 `xhs-cards/template.html`
3. 分析用户输入的文章或项目
4. 按语气原则把技术词换成生活语言
5. 替换 template.html 中所有 `{{PLACEHOLDER}}`
6. 输出到 `output/{slug}-xhs-cards.html`
